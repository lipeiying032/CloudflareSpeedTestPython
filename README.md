## 简介

国内用户访问使用 Cloudflare CDN 的网站时，由于 Cloudflare 分配的 IP 线路质量参差不齐，经常出现延迟高、丢包多、速度慢的情况。本工具通过批量测试 Cloudflare 全部公开 IP 段的延迟和下载速度，帮助你找到当前网络环境下质量最优的 IP。

**Python 移植版的优势：**

- 单文件，无需编译，`python3 cfst.py` 直接运行
- 纯标准库实现，零额外依赖（彩色输出可选安装 `colorama`）
- 跨平台：Windows / Linux / macOS / Android Termux 均可使用
- 与原版 Go 实现在算法和输出格式上保持一致

## 快速开始

### 1. 准备文件

将以下三个文件放在同一目录下：

```
cfst.py        ← 主脚本
ip.txt         ← Cloudflare IPv4 段（见下方 IP 池说明）
ipv6.txt       ← Cloudflare IPv6 段（可选）
```

### 2. 运行（默认参数）

```bash
# Linux / macOS
python3 cfst.py

# Windows
python cfst.py

# Android Termux
python cfst.py
```

默认会对 `ip.txt` 中的 **5955 个 IPv4 地址**（每个 /24 随机取 1 个）进行 TCPing 延迟测速，再对延迟最低的前 10 个进行下载测速，结果保存到 `result.csv`。

### 3. 查看结果

测速完成后，终端会打印最优的 10 个 IP：

```
IP 地址           已发送  已接收  丢包率  平均延迟  下载速度(MB/s)  地区码
104.27.200.69     4      4       0.00   146.23    28.64          LAX
172.67.60.78      4      4       0.00   139.82    15.02          SEA
...
```

完整结果保存在 `result.csv`，用记事本或表格软件打开即可。

## 完整参数说明

```
python3 cfst.py -h
```

### 基础测速参数

| 参数 | 默认值 | 说明 |
|---|---|---|
| `-n 200` | 200 | 并发线程数，最多 1000。路由器等弱设备建议调低到 50 以下 |
| `-t 4` | 4 | 单个 IP 的延迟测速次数 |
| `-dn 10` | 10 | 下载测速的 IP 数量（从延迟最低开始） |
| `-dt 10` | 10 | 单个 IP 下载测速的最长时间（秒），不能太短 |
| `-tp 443` | 443 | 测速使用的端口（延迟测速和下载测速共用） |
| `-url URL` | cf.xiu2.xyz/url | 下载测速地址，建议自建以保证稳定性 |

### 测速模式参数

| 参数 | 说明 |
|---|---|
| `-httping` | 切换为 HTTP 延迟测速模式（默认为 TCPing）。服务器上运行时需降低 `-n`，避免被识别为扫描 |
| `-httping-code 200` | HTTPing 模式下认为成功的 HTTP 状态码，只能指定一个（默认接受 200/301/302） |
| `-cfcolo HKG,LAX` | 仅保留指定地区的 IP，逗号分隔，**仅 HTTPing 模式可用**。Cloudflare/CloudFront/Fastly 用三字机场码（如 HKG），CDN77/Bunny 用二字国家码（如 US），Gcore 用二字城市码 |

### 过滤条件参数

| 参数 | 默认值 | 说明 |
|---|---|---|
| `-tl 200` | 9999 | 平均延迟上限（ms），超过此值的 IP 会被过滤 |
| `-tll 40` | 0 | 平均延迟下限（ms），低于此值的 IP 会被过滤 |
| `-tlr 0.2` | 1.00 | 丢包率上限（0.00~1.00），`-tlr 0` 过滤掉所有有丢包的 IP |
| `-sl 5` | 0.00 | 下载速度下限（MB/s），需凑够 `-dn` 数量才停止测速 |

### 输入输出参数

| 参数 | 默认值 | 说明 |
|---|---|---|
| `-f ip.txt` | ip.txt | IP 段数据文件路径 |
| `-ip 1.1.1.1,2.2.2.2/24` | 空 | 直接通过参数指定 IP 段，逗号分隔 |
| `-o result.csv` | result.csv | 结果文件路径，`-o ""` 禁用文件输出 |
| `-p 10` | 10 | 终端显示的结果数量，`-p 0` 不显示直接退出 |

### 其他参数

| 参数 | 说明 |
|---|---|
| `-dd` | 禁用下载测速，结果按延迟排序（不再按速度排序） |
| `-allip` | 测速 IP 段内的**全部** IPv4 地址（默认每个 /24 只随机测一个） |
| `-debug` | 调试模式，输出详细报错信息，排查下载速度为 0.00 时很有用 |
| `-v` | 打印版本号并检查是否有 Go 原版更新 |
| `-h` | 打印帮助说明 |

## 使用示例

### 基础用法

```bash
# 默认参数运行（最常用）
python3 cfst.py

# 仅做延迟测速，不下载测速（速度更快）
python3 cfst.py -dd

# 限制延迟上限 200ms，同时要求下载速度 ≥ 5 MB/s，找够 10 个
python3 cfst.py -tl 200 -sl 5 -dn 10

# 用 HTTPing 模式，只保留香港节点
python3 cfst.py -httping -cfcolo HKG

# 指定 IPv6 文件测速
python3 cfst.py -f ipv6.txt

# 直接指定 IP 进行测速
python3 cfst.py -ip 1.1.1.1,1.0.0.0/24,2606:4700::/32

# 测 80 端口（需指定 http:// 地址）
python3 cfst.py -tp 80 -url http://cdn.cloudflare.steamstatic.com/steam/apps/5952/movie_max.webm

# 不输出结果文件，只在终端显示
python3 cfst.py -o "" -p 10

# 增加线程数加速测速（网络好的机器）
python3 cfst.py -n 500

# 路由器等弱设备，降低并发
python3 cfst.py -n 30 -t 2

# 全量测速所有 IP（数量极多，时间较长）
python3 cfst.py -allip -dd
```

### 进阶：搭配条件使用建议

```bash
# 先不带条件跑一遍，了解当前网络的大致延迟和速度范围
python3 cfst.py

# 根据结果再加条件过滤，例如延迟 < 150ms 且速度 > 3 MB/s
python3 cfst.py -tl 150 -sl 3 -dn 5

# 如果一直找不够 -dn 指定数量，加 -debug 看看实际速度范围，再调低 -sl
python3 cfst.py -tl 150 -sl 3 -dn 5 -debug
```

> **提示**：同时指定 `-sl` 和 `-tl` 是最佳实践。单独指定 `-sl` 但不指定 `-tl`，当可用 IP 很多时，程序会一直测速下去直到凑够数量，可能耗费大量时间。

## 结果说明

### 终端输出示例

```
IP 地址           已发送  已接收  丢包率  平均延迟  下载速度(MB/s)  地区码
104.27.200.69     4      4       0.00   146.23    28.64          LAX
172.67.60.78      4      4       0.00   139.82    15.02          SEA
104.25.140.153    4      4       0.00   146.49    14.90          SJC
```

### 各列含义

| 列名 | 说明 |
|---|---|
| IP 地址 | 被测试的 Cloudflare IP |
| 已发送 | TCPing/HTTPing 发送的探测包数量（`-t` 参数控制） |
| 已接收 | 成功收到响应的次数 |
| 丢包率 | `(发送 - 接收) / 发送`，0.00 表示无丢包 |
| 平均延迟 | 所有成功探测的平均响应时间（毫秒） |
| 下载速度(MB/s) | 下载测速的平均速度，`-dd` 禁用时显示 0.00 |
| 地区码 | 该 IP 当前服务的数据中心位置（需 HTTPing 或下载测速时才能获取） |

### 排序规则

- **启用下载测速（默认）**：按下载速度从高到低排序
- **禁用下载测速（`-dd`）**：按丢包率升序，丢包率相同则按延迟升序

### CSV 文件

结果默认写入 `result.csv`（UTF-8 with BOM 编码，Excel 可直接正常显示中文）。

## 测速原理

### TCPing（默认模式）

直接对目标 IP 的指定端口（默认 443）发起 TCP 连接，记录连接耗时作为延迟。超时时间为 1 秒。速度快，资源占用低。

### HTTPing 模式（`-httping`）

发送 HEAD 请求到测速 URL，根据 HTTP 响应状态码判断可用性，同时从响应头中提取地区码。超时时间为 2 秒。比 TCPing 更接近真实访问体验，但资源消耗更大，在服务器上运行时需降低并发。

同一 IP 不同协议的延迟大小关系通常为：ICMP < TCP < HTTP

### 下载测速

向测速 URL 发起 GET 请求，下载文件内容并记录速度。支持跟随 HTTP 重定向（最多 10 次），TCP 连接始终打到被测速的 IP 上（不经 DNS 解析）。

使用 **EWMA（指数加权移动平均）** 算法计算实时速度，与 Go 原版采用相同的 `alpha ≈ 0.125`（age=15）参数，最终速度公式为 `ewma_value / (timeout / 120)`。

### 地区码来源

| CDN | 响应头 | 地区码格式 |
|---|---|---|
| Cloudflare | `cf-ray` | IATA 三字机场码（如 HKG、LAX） |
| AWS CloudFront | `x-amz-cf-pop` | IATA 三字机场码 |
| Fastly | `x-served-by` | IATA 三字机场码（取最后一个） |
| CDN77 | `x-77-pop` | 二字国家码（如 US、CN） |
| Bunny CDN | `server` | 二字国家码 |
| Gcore | `x-id-fe` | 二字城市码 |

## 常见问题排查

### 下载速度全部为 0.00

这是最常见的问题，几乎都与网络或地址有关，与脚本本身无关。

**第一步：开启调试模式，看具体报错原因**

```bash
python3 cfst.py -debug
```

**常见报错及原因：**

| 报错关键词 | 原因 |
|---|---|
| `Connection reset by peer` | 下载测速地址被阻断，或该 IP 被针对性封锁 |
| `HTTP 状态码: 403` | 测速地址拒绝访问 |
| `HTTP 状态码: 404` | 测速地址文件不存在 |
| `context deadline` / `timed out` | 请求超时，IP 或网络问题，也可能 `-dt` 时间太短 |
| `certificate` / `tls` | TLS 握手失败，IP 与测速地址的 CDN 不匹配 |

**第二步：换一个测速地址试试**

默认测速地址可用性不保证，可以尝试其他地址（参考原项目 Discussions #490）或自建：

```bash
# 换一个测速地址
python3 cfst.py -url https://你的自建地址/大文件.bin
```

**第三步：换一套 IP 段试试**

部分 IP（Cloudflare 回源 IP）本身不可用于下载测速。可以加 `-sl 0.01` 过滤掉速度极低的结果，先了解整体速度范围。

### 平均延迟非常低（如 0.xx ms）

说明测速时走了代理，请先关闭代理软件或将测速程序排除在代理规则之外，再重新测速。

### 在路由器上运行结果不准确

需要将路由器内的代理软件对本机 TCP 连接设置为直连（或直接关闭），否则延迟测速结果会偏低（实际走的是代理延迟而非真实线路延迟）。

### HTTPing 可用 IP 越来越少直到为 0

可能被运营商或 Cloudflare 识别为网络扫描，触发临时限制，过一段时间会自动恢复。解决方法：降低并发数 `-n`。

### 找不到足够数量的满足条件的 IP

条件可能设置过严。建议先不带任何过滤条件跑一次，查看实际的延迟和速度分布，再适当调整 `-tl`、`-sl` 的值。

### Windows 乱码（CSV 文件）

`result.csv` 使用 UTF-8 with BOM 编码，Excel 可以正确打开。如果用旧版 Excel 仍然乱码，可以在 Excel 中选择"数据 → 从文本/CSV 导入"并手动指定 UTF-8 编码。

## 与原版的已知差异

| 项目 | Go 原版 | Python 移植版 | 影响 |
|---|---|---|---|
| 并发模型 | goroutine（协程） | threading（系统线程） | Python 线程有 GIL，高并发（>500）时 CPU 效率略低于 Go，但网络 I/O 不受影响 |
| 进度条库 | cheggaaa/pb | 纯标准库实现 | 视觉效果略有差异，功能一致 |
| 彩色输出 | fatih/color | colorama（可选） | 未安装时退化为纯文本，功能不受影响 |
| 版本号 | Go 编译时注入 | `v1.0.0-py` | 版本检查会提示 Go 原版的新版本 |
| IPv6 采样 | 完全随机 | 最多 128 个/段 | 超大 IPv6 段的覆盖率略低，可接受 |
